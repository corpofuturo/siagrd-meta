### Gotcha de infraestructura nuevo (no estaba documentado)

`deploy.yml` no maneja el escenario de un `docker compose up -d --build` interrumpido a medio camino (p. ej. por timeout del SSH runner) — deja contenedores en estado `Dead`/`Created` con nombre en conflicto y el stack no vuelve a levantar solo. Si vuelve a pasar:

```
docker ps -a                 # buscar contenedores Dead/Created con nombre en conflicto
docker rm -f <esos contenedores>
docker compose --env-file .env up -d
```

No se implementó manejo automático de este caso en `deploy.yml` — deuda de infraestructura, no bloqueante, delegar en `infra-hardening` si se quiere blindar el workflow.
